1. Peringatan Keselamatan Kerja (K3)

PERHATIAN! KONFIGURASI KRITIS & MODULAR
  • Platform AVP sering bersifat modular. Pastikan Anda mengkonfigurasi modul/slot yang benar.
  • Kesalahan konfigurasi (PID conflict, bitrate) bisa menyebabkan siaran mati total.
  • Selalu simpan ('Save'/'Commit') konfigurasi setelah melakukan perubahan.

2. Prosedur Koneksi Awal

  1. Hubungkan kabel input stream (ASI atau IP) dari Encoder ke port input pada modul MUX di chassis AVP.
  2. Hubungkan kabel output (ASI atau IP) dari port output modul MUX ke Pemancar.
  3. Hubungkan kabel LAN (RJ45) dari port 'Management' chassis AVP ke Jaringan LAN/Laptop.
  4. Nyalakan chassis AVP. Tunggu hingga semua modul selesai booting.

3. Prosedur Konfigurasi (WebGUI)

IP Address chassis AVP harus diketahui (Contoh: `192.168.1.11`).

  1. Atur IP Address Laptop/PC Anda ke segmen yang sama.
  2. Buka Browser dan ketik IP Address Chassis (misal: `http://192.168.1.11`).
  3. Masuk menggunakan username dan password.
  4. Pilih Modul MUX:
    • Navigasi ke slot tempat modul MUX terpasang.
  5. Konfigurasi Input Stream:
    • Cari menu `Input Configuration` atau `Stream Input`.
    • Pilih port input (misal: ASI 1) dan aktifkan. Pastikan status 'Link Up' atau 'Signal OK'.
    • Ulangi untuk input lain jika ada (misal: ASI 2).
  6. Membuat Output Transport Stream:
    • Cari menu `Multiplexing` atau `Output TS Configuration`.
    • Buat TS output baru. Atur TS ID dan Network ID.
  7. Mapping Services (Program):
    • Pilih TS output yang baru dibuat. Masuk ke 'Service Mapping' atau 'PID Filtering/Remapping'.
    • Pilih 'Input Port' (misal: ASI 1).
    • Pilih Service Name atau Program Number yang ingin dimasukkan.
    • Atur PID Output (Video, Audio, PMT) jika perlu di-remap.
    • Ulangi untuk Service dari input lain (misal: ASI 2).
  8. Konfigurasi PSI/SI Tables:
    • Masuk ke menu `PSI/SI Generation`.
    • Aktifkan pembuatan PAT, PMT, SDT, NIT. Atur parameter (Nama Service, Provider, Network Name) dan interval.
  9. Konfigurasi Output Fisik:
    • Masuk ke menu `Output Configuration`.
    • Pilih port fisik output (misal: ASI Out 1).
    • Pilih 'Source' = TS output yang tadi dibuat.
    • Atur 'Output Mode' (misal: CBR) dan 'Bitrate'.
  10. Klik 'Save Configuration' atau 'Commit Changes'.

5. Troubleshooting

Apa masalah Anda?

Solusi: Cek Output Config & Bitrate

Pastikan `Output Configuration` sudah benar memilih 'Source' TS Output. Cek juga apakah total bitrate service melebihi batas 'Bitrate' output yang diatur.

Solusi: Cek Input & Service Mapping

Pastikan Input Stream sumber service tersebut statusnya OK. Lalu cek di `Service Mapping` atau `PID Filtering`, pastikan service tersebut sudah ditambahkan dan PID-nya ter-mapping dengan benar.

Solusi: Cek Konfigurasi PSI/SI

Masuk ke `PSI/SI Generation`. Pastikan tabel PAT dan PMT aktif dan semua PID service sudah tercantum. Cek juga interval pengiriman.
